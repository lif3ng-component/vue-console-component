<template>
  <div>
    <div>
      <nq-switch v-model="v1" />
      value: {{ v1 }}
      <nq-button @click="toggleValue">toggle value</nq-button>
    </div>
    <div>
      <nq-switch v-model="v2" :disabled="v2Disabled" />
      disabled: {{ v2Disabled }}
      <nq-button @click="toggleDisalbed">toggle disalbed</nq-button>
    </div>
    <div>
      <nq-switch v-model="v3" :loading="v3Loading" />
      loading: {{ v3Loading }}
      <nq-button @click="toggleLoading">toggle loading</nq-button>
    </div>
  </div>
</template>
<script>
export default {
  data() {
    return {
      v1: true,
      v2: true,
      v2Disabled: true,
      v3: true,
      v3Loading: true
    };
  },
  methods: {
    toggleValue() {
      this.v1 = !this.v1;
    },
    toggleDisalbed() {
      this.v2Disabled = !this.v2Disabled;
    },
    toggleLoading() {
      this.v3Loading = !this.v3Loading;
    }
  }
};
</script>
<style lang="stylus" scoped>
div {
  user-select: none;
}
</style>

# props

# v-model

  • Type: boolean
  • Default: false

# disabled

  • Type: boolean
  • Default: false

显示为 disabled 状态，此时无法点击

# loading

  • Type: boolean
  • Default: false

显示为 loading 状态，此时呈现 disabled 状态并显示 loading 图标，无法点击。

设置为 true 时，disabled 属性无效

# togglePromiseFn

  • Type: function
  • Default: null

切换状态是运行此函数，参数为将要变成的状态(true/false)

  • 返回值必须是一个 promise.
    • pending 展示 loading 图标
    • fulfilled 成功切换
    • rejected 未成功切换。处理异常时，注意保持 rejected 状态以便 switch 正确呈现

设置后，disabled 和 loading 属性无效
